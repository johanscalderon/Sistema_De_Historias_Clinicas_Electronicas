Prueba del patrón Factory Method

Esta pantalla envía una observación médica al backend. El valor de deviceType le indica a la fábrica qué handler usar: heartrate, temperature o pressure.

Prueba del Factory Method: el backend recibe deviceType y la DeviceHandlerFactory decide qué handler usar. Luego el handler concreto procesa y guarda la observación.

Si tu entidad Observation en el backend pide otro campo adicional, agrégalo aquí. Este frontend está pensado para el flujo básico del UML.
